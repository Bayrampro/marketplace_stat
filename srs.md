# Техническое задание  
## на разработку веб-приложения «МаркетплейсСтат»

---

## 1. Общие сведения

### 1.1. Наименование проекта

Веб-приложение **«МаркетплейсСтат»** — сайт-помощник для селлеров маркетплейса **Wildberries**.

### 1.2. Основание для разработки

Разработка выполняется в рамках дипломного проекта на тему:

**«Разработка веб-приложения на примере “МаркетплейсСтат” для помощи селлерам Wildberries»**.

### 1.3. Назначение системы

Веб-приложение предназначено для автоматизации отдельных задач селлера Wildberries, связанных с подготовкой карточки товара, созданием визуального контента и анализом конкурентной среды.

Система должна предоставлять пользователю три основные функции:

1. создание SEO-описания товара на основе ключевых слов;
2. генерация инфографики для главной фотографии товара;
3. подбор товаров-конкурентов по ключевой фразе на основе тестовых данных.

### 1.4. Цель разработки

Целью разработки является создание веб-приложения, которое позволяет селлеру Wildberries быстрее подготавливать карточки товаров и получать вспомогательную информацию для анализа конкурентной выдачи.

### 1.5. Основные пользователи системы

| Пользователь | Описание |
|---|---|
| Селлер Wildberries | Использует систему для подготовки карточек товаров |
| Начинающий продавец | Получает помощь при создании описаний и инфографики |
| Администратор | Управляет тестовыми данными и контролирует работу приложения |

## 2. Общая характеристика системы

Веб-приложение **«МаркетплейсСтат»** должно иметь простой интерфейс с тремя основными вкладками.

| Вкладка | Назначение |
|---|---|
| SEO-описание | Генерация текстового описания товара |
| Инфографика | Создание изображения с графическими и текстовыми элементами |
| Конкуренты | Поиск товаров-конкурентов по ключевой фразе |

Приложение разрабатывается как демонстрационная MVP-версия для дипломного проекта. Основной акцент делается на работоспособности, понятной архитектуре и возможности показать все функции без зависимости от платных API и нестабильных внешних сервисов.

## 4.1. Функция 1. Создание SEO-описания товара

### 4.1.1. Назначение функции

Функция предназначена для автоматического формирования SEO-описания товара на основе введенных пользователем данных: названия товара, категории, ключевых слов и преимуществ.

### 4.1.2. Используемые инструменты

| Инструмент | Назначение |
|---|---|
| Django | Обработка формы и вывод результата |
| Python-шаблоны | Формирование структуры описания |
| pymorphy3 | Морфологическая обработка ключевых слов |
| random | Выбор разных вариантов шаблонов |
| re | Очистка и обработка введенного текста |

### 4.1.3. Входные данные

| Поле | Описание | Пример |
|---|---|---|
| Название товара | Основное название товара | Женская черная шапка |
| Категория товара | Категория, к которой относится товар | Головные уборы |
| Ключевые слова | SEO-фразы через запятую | черная шапка, зимняя шапка, женская шапка |
| Преимущества товара | Основные характеристики товара | теплая, мягкая, универсальный размер |

### 4.1.4. Алгоритм работы функции

1. Пользователь открывает вкладку **«SEO-описание»**.
2. Заполняет поля формы.
3. Нажимает кнопку **«Сгенерировать описание»**.
4. Система получает введенные данные.
5. Ключевые слова очищаются от лишних пробелов и повторов.
6. С помощью `pymorphy3` слова приводятся к начальной форме для выявления смысловых дублей.
7. Система выбирает один из заранее подготовленных текстовых шаблонов.
8. В шаблон подставляются название товара, категория, ключевые слова и преимущества.
9. Готовое SEO-описание отображается на странице.
10. Пользователь может скопировать полученный текст.

### 4.1.5. Требования к генерации текста

Система должна:

- формировать связный текст описания;
- использовать ключевые слова в естественной форме;
- исключать чрезмерные повторы одинаковых слов;
- учитывать название товара и его преимущества;
- генерировать разные варианты описаний при повторном запуске;
- отображать результат в отдельном блоке;
- предоставлять возможность копирования текста.

### 4.1.6. Пример результата

Пользователь вводит:

```text
Название товара: Женская черная шапка
Ключевые слова: черная шапка, зимняя шапка, женская шапка
Преимущества: теплая, мягкая, универсальный размер
```

Система формирует:

> Женская черная шапка — практичный аксессуар для холодного времени года. Модель подходит для повседневной носки, прогулок и создания зимних образов. Изделие отличается мягким материалом, удобной посадкой и универсальным размером. Такая зимняя шапка станет хорошим выбором для покупателей, которые ценят комфорт, тепло и лаконичный внешний вид.

## 4.2. Функция 2. Генерация инфографики для главной фотографии товара

### 4.2.1. Назначение функции

Функция предназначена для создания инфографики на основе изображения товара, загруженного пользователем. Система должна накладывать на изображение текстовые и графические элементы, которые выделяют преимущества товара.

### 4.2.2. Используемые инструменты

| Инструмент | Назначение |
|---|---|
| Django File Upload | Загрузка изображения пользователем |
| Pillow | Обработка изображения и наложение графики |
| random | Случайный выбор оформления |
| Django Media | Хранение исходных и готовых изображений |
| HTML/CSS/JS | Предпросмотр результата на странице |
| Download file | Скачивание готовой инфографики |

### 4.2.3. Входные данные

| Поле | Описание | Пример |
|---|---|---|
| Изображение товара | Фото в формате JPG, PNG или WEBP | product.jpg |
| Заголовок | Главный текст на изображении | Теплая зимняя шапка |
| Преимущество 1 | Первое преимущество товара | Мягкий материал |
| Преимущество 2 | Второе преимущество товара | Защита от холода |
| Преимущество 3 | Третье преимущество товара | Универсальный размер |

### 4.2.4. Алгоритм работы функции

1. Пользователь открывает вкладку **«Инфографика»**.
2. Загружает изображение товара.
3. Вводит заголовок и преимущества товара.
4. Нажимает кнопку **«Создать инфографику»**.
5. Система проверяет формат и размер файла.
6. Исходное изображение сохраняется в каталог `media/uploads/`.
7. Библиотека Pillow открывает изображение.
8. Система приводит изображение к нужному размеру.
9. Случайным образом выбирается вариант компоновки.
10. Случайным образом выбирается цветовая схема и расположение элементов.
11. На изображение накладываются заголовок, плашки, рамки и преимущества.
12. Готовое изображение сохраняется в каталог `media/generated/`.
13. Пользователь видит предпросмотр результата.
14. Пользователь может скачать готовую картинку.

### 4.2.5. Требования к оформлению инфографики

Инфографика не должна создаваться всегда по одному шаблону. Необходимо реализовать **контролируемую рандомизацию оформления**.

Случайно могут выбираться:

| Элемент | Возможные варианты |
|---|---|
| Расположение заголовка | сверху, снизу, слева |
| Расположение преимуществ | справа, слева, снизу, по углам |
| Форма плашек | прямоугольные, скругленные, круглые |
| Цветовая схема | светлая, контрастная, пастельная |
| Декоративные элементы | рамки, линии, маркеры, тени |
| Размер текста | средний или крупный |

При этом рандомизация должна быть управляемой, чтобы итоговое изображение оставалось читаемым и аккуратным.

### 4.2.6. Требования к загрузке файла

Система должна:

- принимать изображения форматов JPG, PNG, WEBP;
- ограничивать размер загружаемого файла;
- проверять наличие файла перед генерацией;
- не допускать загрузку исполняемых файлов;
- сохранять исходное изображение отдельно от готового результата.

### 4.2.7. Требования к скачиванию результата

После генерации инфографики система должна:

- сохранить готовую картинку в папку `media/generated/`;
- показать пользователю предпросмотр;
- отобразить кнопку **«Скачать изображение»**;
- позволить пользователю загрузить итоговый файл на устройство.

Формат готового файла:

```text
generated_product_image.png
```

### 4.2.8. Пример результата работы функции

Пользователь загружает фото товара и вводит:

```text
Заголовок: Теплая зимняя шапка
Преимущество 1: Мягкий материал
Преимущество 2: Защита от холода
Преимущество 3: Универсальный размер
```

Система генерирует изображение, на котором размещены:

- заголовок товара;
- несколько текстовых плашек;
- декоративные рамки;
- преимущества товара;
- визуальное оформление, выбранное случайным образом.

## 4.3. Функция 3. Подбор товаров-конкурентов по ключевой фразе

### 4.3.1. Назначение функции

Функция предназначена для демонстрации подбора товаров-конкурентов по ключевой фразе. В рамках дипломного проекта данные о товарах-конкурентах хранятся в базе данных SQLite.

Система не получает реальные продажи конкурентов Wildberries и не использует закрытые данные маркетплейса. Функция моделирует работу анализа конкурентной выдачи на основе заранее подготовленного набора данных.

### 4.3.2. Используемые инструменты

| Инструмент | Назначение |
|---|---|
| SQLite | Хранение тестовых данных о товарах |
| Django ORM | Поиск и выборка товаров |
| HTML/CSS | Вывод карточек или таблицы конкурентов |
| JavaScript | Улучшение интерфейса поиска |

### 4.3.3. Входные данные

| Поле | Описание | Пример |
|---|---|---|
| Ключевая фраза | Поисковый запрос для подбора конкурентов | черная шапка |

### 4.3.4. Структура тестовых данных

В базе данных должна быть таблица товаров-конкурентов.

| Поле | Тип | Описание |
|---|---|---|
| id | Integer | Уникальный идентификатор |
| search_query | Text | Ключевая фраза |
| title | Text | Название товара |
| price | Integer | Цена товара |
| rating | Float | Рейтинг товара |
| reviews_count | Integer | Количество отзывов |
| image | Text / ImageField | Изображение товара |
| product_url | Text | Ссылка на товар |
| position | Integer | Позиция в условной выдаче |

### 4.3.5. Алгоритм работы функции

1. Пользователь открывает вкладку **«Конкуренты»**.
2. Вводит ключевую фразу.
3. Нажимает кнопку **«Найти конкурентов»**.
4. Django получает запрос пользователя.
5. Система выполняет поиск по таблице товаров.
6. Если найдено точное совпадение по `search_query`, выводятся соответствующие товары.
7. Если точного совпадения нет, выполняется частичный поиск по словам.
8. Пользователю отображается до 5 товаров-конкурентов.
9. Если товары не найдены, система выводит сообщение об отсутствии данных.

### 4.3.6. Выходные данные

Система должна выводить список до 5 товаров.

Для каждого товара отображаются:

| Данные | Обязательность |
|---|---|
| Название товара | Да |
| Цена | Да |
| Рейтинг | Да |
| Количество отзывов | Да |
| Изображение | Да |
| Ссылка на товар | Да |
| Позиция в выдаче | Да |

### 4.3.7. Пример результата

Пользователь вводит:

```text
черная шапка
```

Система выводит:

| № | Название товара | Цена | Рейтинг | Отзывы |
|---|---|---:|---:|---:|
| 1 | Шапка женская зимняя черная | 599 ₽ | 4.8 | 1250 |
| 2 | Шапка бини черная | 749 ₽ | 4.7 | 870 |
| 3 | Шапка утепленная базовая | 690 ₽ | 4.9 | 1430 |
| 4 | Женская шапка с отворотом | 520 ₽ | 4.6 | 640 |
| 5 | Зимняя шапка универсальная | 799 ₽ | 4.8 | 980 |

### 4.3.8. Формулировка ограничения

В дипломной работе необходимо указать:

> В демонстрационной версии проекта модуль подбора конкурентов работает с тестовым набором данных, сохраненным в SQLite. Такой подход позволяет показать принцип поиска и анализа товаров-конкурентов без подключения к внешним сервисам и без использования закрытых данных Wildberries.

## 9. Требования к безопасности

- не сохранять исполняемые файлы;

- не использовать закрытые данные Wildberries.

## 10. Ожидаемый результат разработки

В результате разработки должно быть создано веб-приложение **«МаркетплейсСтат»**, включающее три функциональных модуля:

### 10.1. Модуль SEO-описания

Позволяет сформировать описание товара на основе названия, категории, ключевых слов и преимуществ. Для обработки ключевых слов используется библиотека `pymorphy3`.

### 10.2. Модуль генерации инфографики

Позволяет загрузить изображение товара, добавить заголовок и преимущества, сгенерировать случайно оформленную 2D-инфографику с помощью Pillow, просмотреть результат и скачать готовую картинку.

### 10.3. Модуль подбора конкурентов

Позволяет найти до пяти товаров-конкурентов по ключевой фразе на основе тестового набора данных, сохраненного в SQLite.

Разработанное приложение должно демонстрировать принцип работы сервиса-помощника для селлера Wildberries и может быть расширено в дальнейшем за счет подключения внешних API, личного кабинета, истории запросов и реального анализа поисковой выдачи.

## 11. Краткая итоговая формулировка для диплома

Веб-приложение **«МаркетплейсСтат»** разрабатывается на основе Django, HTML, CSS и JavaScript с использованием базы данных SQLite.

Система включает три основные функции:

1. генерацию SEO-описания товара с применением шаблонов и библиотеки `pymorphy3`;
2. создание инфографики на основе загруженного изображения с помощью Pillow и контролируемой рандомизации оформления;
3. подбор товаров-конкурентов по ключевой фразе на основе тестовых данных.

Такой подход позволяет создать стабильную демонстрационную версию проекта без зависимости от платных API и нестабильных внешних сервисов.
